
## **LLd --> Low Level Design**

- Low Level is basically refers to the practice that must be followed while building any application
- LLD mainly focuses on 3 components
    - Scalability --> It means how well our application is handling millions of users and how easily we can add new features on our application and scale it.
    - Maintainability --> It ensures that when we integrate any new feature the previous state of our application works fine and doesn't crash.
    - Reusablity --> It means that the features which are not application depenedent like payment gateway, Notification Manager can be easily implemented on other application as well. Inshort our application should not be tightly coupled.

- LLD deals with the structure of code.


---
## **HLD --> High Level Design**

- HLD or High Level System Design mainly focuses on the architecture of the system.
- What **Tech Stack** we are using 
- What **Database** we are using whether its SQL or NoSQL
- How we're scaling the server for large user traffic.
- How we're optimizing the cost 

---
## **DSA --> Data Structures & Algorithm**

- DSA handles the actual logic part of the application.
- It's the actual brain of any application

---

**While building any application DSA, LLD, HLD all three are used.**
